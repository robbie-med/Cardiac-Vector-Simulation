         - LVH: Increases LV mass 1.5×, wall thickness 1.7×, APD +25ms (tall LV, deep RV)
         - RVH: Increases RV mass 1.8×, wall thickness 2.0×, APD +15ms (right axis)
         - LAE: Increases LA mass 1.8×, reduces conduction velocity 0.7×, APD +20ms
         - RAE: Increases RA mass 1.8× (tall P-wave)
       - Ischemia Settings:
         - Occluded segments: health → 0.25 (reduced), injuryCurrent → 0.8, APD +30ms
         - Artery territories: LAD, D1, LCx, OM, RCA, PDA
       - Imports: MyocardialSegment, ConductionSystem types
       - Used By: simulation-cache.ts, ControlPanel, AboutModal

       /src/engine/action-potential.ts

       - Purpose: Computes dipole contribution of myocardial segments using dual-bell model
       - Exports:
         - segmentDipole(segment, activationTime, cycleTimeMs): Vec3
         - computeNetDipole(segments, activationTimes, cycleTimeMs): Vec3 (sum all segments)
       - Dipole Model:
         - Dual-bell formulation: depolarization bell + repolarization bell
         - Bell function = sigmoid derivative (normalized to peak=1.0)
         - Depolarization: centered at activation time, width = tauRise
         - Repolarization: centered at activation time + APD, width = tauFall
         - Repolarization sign: negative for atria (discordant), positive for ventricles (concordant)
         - Injury current: DC offset in resting segments (away from both bells)
       - Amplitude:
         - Dipole = mass × wallThickness × health × fiberDirection × signal × GAIN
         - GAIN = 0.0476 (calibrated for Lead II R-wave ≈ 1.0 mV)
         - REPOL_FACTOR = 0.10 (T-wave amplitude relative to QRS)
         - INJURY_AMPLITUDE = 0.002 (ischemia DC offset)
       - Imports: Vec3, MyocardialSegment types
       - Used By: Rendering layers, axis-calculator, debug script

       /src/engine/conduction-graph.ts

       - Purpose: Models cardiac conduction system as a directed graph
       - Exports:

         - ConductionSystem interface (edges array)
         - buildDefaultConductionSystem(): ConductionSystem
         - cloneConductionSystem(sys): ConductionSystem (deep clone)
       - Graph Structure:
         - SA node → atrial segments (0, 25, 35, 50 ms)

         - LPF → LV segments (28–70 ms, Purkinje)
         - RBB → RV segments (27–50 ms, Purkinje)

         - Cell-to-cell backup pathways (30–40 ms, for BBB compensation)
       - Timing Reference:
         - SA→AV total: 130 ms
         - AV→His: 10 ms
         - His→BB→fascicle: 10 ms total
         - QRS onset (first ventricular): ~160 ms from SA
         - Total QRS: ~80 ms
         - PR interval: ~120–160 ms
         - P-wave: ~50–100 ms (activation + sigmoid tails)

       - Used By: simulation-cache, pathology

       /src/engine/activation-solver.ts

       - Purpose: Solves activation times for all segments via Dijkstra shortest-path
       - Exports:
         - ActivationMap interface (activationTimes map, phaseBoundaries)
         - PhaseBoundaries interface (pStart, pEnd, qrsStart, qrsEnd, tStart, tEnd, cycleLength)
         - solveActivationTimes(conduction, segments, heartRateBpm): ActivationMap
         - lookupPhase(phaseBoundaries, cycleTimeMs): {phase, phaseT}
       - Algorithm:
         - Build adjacency list from non-blocked edges
         - Dijkstra from 'sa-node' to find shortest path to each segment
         - Extract activation times for segment nodes only
         - Compute phase boundaries from segment APD + tau values
       - Phase Boundaries:
         - P: min(atrial activation) to max(atrial depol + 5×tauRise)
         - PR: P-wave end to QRS start
         - QRS: min(ventricular activation) to max(ventricular depol + 5×tauRise)
         - ST: QRS end to T-wave start (normally isoelectric)
         - T: min(ventricular repol - 2×tauFall) to max(repol + 3×tauFall)
         - TP: T-wave end to next P-wave
         - cycleLength = 60000 / heartRateBpm (ms)
       - Phase Lookup:
         - Returns current phase and normalized progress (0–1) within phase
       - Imports:
         - ConductionSystem, MyocardialSegment, CardiacPhase types
       - Used By: simulation-cache

       - Purpose: Standalone Node.js diagnostic script for ECG analysis
       - Exports: (none - direct execution)
       - Imports:
         - updateSimulation (simulation-cache)
         - computeNetDipole, segmentDipole (action-potential)
         - computeAllLeadVoltages, computeLeadVectors, etc. (lead-calculator)
       - Diagnostics:

       - Output: Formatted ASCII tables to stdout
       - Used By: Manual testing via npx tsx src/engine/ecg-debug.ts

       KEY ARCHITECTURAL PATTERNS

       1. Layered Architecture:
         - Engine Layer: Physics simulation (segments, conduction, activation, dipoles)
         - Cache Layer: Pre-computes and memoizes expensive calculations
         - Rendering Layer: Real-time animation consuming cached data
         - Store Layer: Zustand for UI state
       2. State Flow:
         - User changes params in ControlPanel → store updates
         - Components subscribed to store re-render
         - updateSimulation() called, cached result returned if config unchanged
         - Per-frame getCardiacVector() sums segment dipoles → leads project vector → canvas renders
       3. Coordinate Systems:
         - Frank/ECG coordinates: cardiac vector math (X+left, Y+inferior, Z+anterior)
         - Three.js scene: visual rendering (X+right, Y+up, Z+forward)
         - vectorToScene() / sceneToVector() bridge conversions
       4. Pathology Model:
         - Before: Pre-drawn Bézier curves swapped per condition
         - Now: Conditions modify segment properties (mass, wall thickness, APD) and conduction edges (block/add)
         - Result: ECG morphology emerges from altered propagation + dipole summation (physics-based)
       5. Performance Optimization:
         - Simulation cache avoids recomputation on unchanged config
         - Dijkstra runs once per config change (not per frame)
         - Per-frame: only dipole summation (34 segments × bell function evaluation)
         - Canvas pre-rendered grid (reused across frames)
         - Ring buffers for scrolling traces (memory-efficient)

       SUMMARY

       This is a sophisticated educational ECG simulator that models cardiac electrophysiology from first principles:

       - 30 ventricular + 4 atrial segments, each with mass, fiber direction, APD, health, injury current
       - Dijkstra-based activation solver on a conduction graph matching Durrer 1970 timing
       - Dual-bell dipole model (depolarization + repolarization current) scaled by segment properties
       - Physics-based pathologies: tissue modifications (mass, APD, conduction velocity) + conduction edge blocking/addition
       - Real-time rendering: 12-lead canvas + 3D VCG visualization + interactive 3D heart model
       - Windows XP-styled UI for nostalgic charm without sacrificing function
       - Fully responsive: electrodes can be placed anywhere on torso, all 12 leads update synchronously

       All leads derive from a single 3D cardiac vector — Einthoven's law holds exactly by construction. The simulator
       demonstrates that realistic ECG morphology emerges naturally from cardiac anatomy and physiology, without resorting to
       pre-recorded or hand-drawn patterns.
